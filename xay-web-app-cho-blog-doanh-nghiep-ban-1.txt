     Plan: Thời Trang Hoa Xuân – Full-Stack Fashion Blog Web App

     Context

     Build a full-stack fashion blog web app for brand "Thời Trang Hoa Xuân"
     (Spring Flower Fashion), targeting women aged 18–36 with a young, dynamic
     style. The app includes a public blog, product showcase, and admin panel.
     Requirements: Next.js frontend, FastAPI backend, PostgreSQL, deployed on
     Railway, with all required files (README.md, CLAUDE.md, PLAN.md,
     conversation export).

     ---
     Tech Stack

     ┌──────────┬───────────────────────────────────────────────────┐
     │  Layer   │                    Technology                     │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Frontend │ Next.js 14 (App Router, TypeScript, Tailwind CSS) │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Backend  │ Python FastAPI + SQLAlchemy                       │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Database │ PostgreSQL (Railway managed)                      │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Auth     │ JWT (jose) + bcrypt                               │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Storage  │ Cloudinary (image uploads)                        │
     ├──────────┼───────────────────────────────────────────────────┤
     │ Deploy   │ Railway (3 services: frontend, backend, postgres) │
     └──────────┴───────────────────────────────────────────────────┘

     ---
     Database Schema (PostgreSQL)

     -- users
     id, email, hashed_password, full_name, is_admin, created_at

     -- categories
     id, name, slug, description

     -- posts (blog)
     id, title, slug, content (text/HTML), excerpt, cover_image_url,
     author_id (FK users), category_id (FK categories),
     is_published, published_at, created_at, updated_at

     -- products
     id, name, slug, description, price, discount_price,
     images (JSONB array of URLs), category_id, tags (JSONB),
     is_featured, is_active, created_at, updated_at

     -- tags (optional M2M with posts)
     id, name, slug

     ---
     Frontend Pages & Features

     Public Pages

     - Homepage – hero banner (brand video/image), featured blog posts, featured
      products, brand tagline, newsletter signup
     - Blog Listing – card grid with category filter, search, infinite scroll
     - Blog Detail – full article, author, share buttons, related posts
     - Products – masonry grid, filter by category/price, hover zoom
     - About – brand story, values, team

     Admin Panel (/admin — JWT protected)

     - Dashboard with stats (total posts, products, views)
     - Blog post editor (rich text / markdown with preview)
     - Product manager (image upload, pricing, toggle active/featured)
     - Category manager

     ---
     UI/UX Design System

     - Colors: Primary #F472B6 (pink-400), Accent #FB7185 (rose-400), Neutral
     #1C1917
     - Typography: Playfair Display (headings) + Inter (body)
     - Style: Clean, minimal, feminine — inspired by Korean/Japandi fashion
     blogs
     - Responsive: Mobile-first, breakpoints sm/md/lg/xl

     ---
     Railway Deployment

     Services

     1. postgres – Railway managed PostgreSQL plugin
     2. backend – FastAPI, Python 3.11, uvicorn app.main:app --host 0.0.0.0 
     --port $PORT
     3. frontend – Next.js, next start, env
     NEXT_PUBLIC_API_URL=https://<backend>.railway.app

     Environment Variables

     # Backend
     DATABASE_URL=postgresql://...  (Railway injects automatically)
     SECRET_KEY=<random 64-char>
     CLOUDINARY_URL=cloudinary://...
     ALLOWED_ORIGINS=https://<frontend>.railway.app

     # Frontend
     NEXT_PUBLIC_API_URL=https://<backend>.railway.app
